LGH.111 Create Hidden navigation button like the one for S13 Passcode button - DESIGN SPEC FINALIZED
========================================
**Purpose:** Toggle button to hide/show Sales Objects section for testing purposes (owned/controlled functionality).

**Button Behavior:**
- Style: Like S13 button (black background, no hover/tooltip, blank label)
- Placement: Before S13 button
- Always visible (never hides with section)

**Toggle States (cycles):**
1) Press 1 (odd) → Section HIDDEN
   a) Hide Köp Objekt label
   b) Hide URL textform & GURL, <<, >, >>, :), D1, DD buttons
   c) Hide Index counter (n/n)
   d) Hide Inköpspris (kr) label & textform
   e) Hide Inköpspris (kr) in print output

2) Press 2 (even) → Section SHOWN¨
   a) Show Köp Objekt label
   b) Show URL textform & GURL, <<, >, >>, :), D1, DD buttons
   c) Show Index counter (n/n)
   d) Show Inköpspris (kr) label & textform
   e) Show Inköpspris (kr) in print output

3) Press 3+ (odd again) → Cycles back to HIDDEN

**Design Decisions (Confirmed):**
- Initial state: HIDDEN on first page load (testing/private mode by default)
- Toggle state persistence: STORED in localStorage (persists across page refresh)
- Data retention: URL/prices REMAIN in localStorage when hidden (re-appear when toggled back on)
- Print coupling: Print state tied to toggle state (hidden on screen = hidden in print)
- Button visibility: Toggle button ALWAYS visible (needed to re-enable hidden section)

**Rationale:**
- Site owned/controlled by user for testing purposes
- Hidden section reduces visual clutter while preserving data
- localStorage keeps data safe for user's own browser
- Persistent toggle state improves UX consistency
